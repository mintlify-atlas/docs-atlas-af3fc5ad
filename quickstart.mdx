## Prerequisites

Before you begin, ensure you have:

<Steps>
  <Step title="Python 3.13+">
    Check your Python version:
    ```bash
    python --version
    ```
    Download Python 3.13+ from [python.org](https://www.python.org/downloads/) if needed.
  </Step>
  
  <Step title="UV Package Manager">
    Install UV (Astral's fast package manager):
    
    <CodeGroup>
    ```bash Windows (PowerShell)
    pip install uv
    ```
    
    ```bash Unix/Linux
    curl -LsSf https://astral.sh/uv/install.sh | sh
    ```
    </CodeGroup>
  </Step>
  
  <Step title="Claude Desktop">
    Download and install [Claude Desktop](https://claude.ai/download) (or another MCP-compatible client).
  </Step>
</Steps>

## Installation

The fastest way to get started is using PyPI with `uvx`:

<Steps>
  <Step title="Locate Configuration File">
    Open Claude Desktop and click on your profile icon, then navigate to **Settings > Developer > Edit Config**.
    
    This will open `claude_desktop_config.json` in your default text editor.
    
    <Note>
      The config file is typically located at:
      ```
      %APPDATA%\Claude\claude_desktop_config.json
      ```
    </Note>
  </Step>
  
  <Step title="Add Windows-MCP Configuration">
    Add the following configuration to your `claude_desktop_config.json`:
    
    ```json
    {
      "mcpServers": {
        "windows-mcp": {
          "command": "uvx",
          "args": [
            "windows-mcp"
          ]
        }
      }
    }
    ```
    
    Save the file.
  </Step>
  
  <Step title="Restart Claude Desktop">
    Close and reopen Claude Desktop for the changes to take effect.
    
    <Warning>
      The first time Windows-MCP starts, it may take 1-2 minutes to install dependencies. If the server times out, simply restart Claude Desktop.
    </Warning>
  </Step>
  
  <Step title="Verify Installation">
    In Claude Desktop, look for the MCP tools icon (🔌) in the chat interface. Click it to see the available Windows-MCP tools.
    
    You should see 15+ tools including App, PowerShell, FileSystem, Snapshot, Click, Type, and more.
  </Step>
</Steps>

## Your First Automation

Now let's run a simple automation to verify everything works:

<Steps>
  <Step title="Take a Snapshot">
    In Claude Desktop, type:
    ```
    Take a snapshot of my desktop
    ```
    
    Claude will use the **Snapshot** tool to capture:
    - Current cursor position
    - Active and background windows
    - Interactive UI elements (buttons, text fields, links) with coordinates
    - Scrollable areas
    
    You should see a detailed text output describing your desktop state.
  </Step>
  
  <Step title="Open Notepad">
    Ask Claude to open an application:
    ```
    Open Notepad
    ```
    
    Claude will use the **App** tool in `launch` mode to open Notepad from the Start menu.
    
    <Note>
      Windows-MCP can launch any application accessible via the Windows Start menu. Just use the application's common name.
    </Note>
  </Step>
  
  <Step title="Type Some Text">
    Now let's type text into Notepad:
    ```
    Type "Hello from Windows-MCP!" in Notepad
    ```
    
    Claude will:
    1. Take a snapshot to find the Notepad text area
    2. Click on it to focus
    3. Use the **Type** tool to enter the text
    
    You should see the text appear in Notepad!
  </Step>
  
  <Step title="Execute a PowerShell Command">
    Let's try a system command:
    ```
    Show me the current date and time using PowerShell
    ```
    
    Claude will use the **PowerShell** tool to execute `Get-Date` and return the result.
  </Step>
</Steps>

## Example: Complete Workflow

Here's a more complex example that demonstrates multiple tools working together:

```text
Create a new text file on my desktop called "tasks.txt" and add the following:
- Review pull requests
- Update documentation
- Fix bug #123
```

Claude will automatically:

1. **Snapshot**: Capture desktop state to understand the environment
2. **FileSystem**: Use the `write` mode to create `tasks.txt` on the desktop with the content
3. **App**: Optionally open the file in Notepad to confirm

You can verify the file was created by checking your desktop.

## Understanding Tool Interactions

Windows-MCP tools work together seamlessly. Here's a typical workflow:

<Steps>
  <Step title="Observe (Snapshot)">
    The **Snapshot** tool is always called first to understand the current desktop state, including:
    - What windows are open
    - What UI elements are available
    - Where elements are located (coordinates)
    - What's scrollable
  </Step>
  
  <Step title="Navigate (Click, Scroll, Move)">
    Based on the snapshot, Claude uses navigation tools to:
    - Click on buttons, links, or menu items
    - Scroll through content
    - Move the mouse to specific locations
  </Step>
  
  <Step title="Input (Type, Shortcut)">
    Input tools allow Claude to:
    - Type text into fields
    - Execute keyboard shortcuts (Ctrl+S, Alt+Tab, etc.)
  </Step>
  
  <Step title="Execute (App, PowerShell, FileSystem)">
    System-level tools enable Claude to:
    - Launch applications
    - Run shell commands
    - Manage files and folders
  </Step>
</Steps>

## Common Patterns

### Web Automation with DOM Mode

```text
Open Chrome, go to https://example.com, and extract all the article titles
```

Claude will:
1. Launch Chrome using **App**
2. Type the URL and press Enter
3. Use **Snapshot** with `use_dom=True` to extract web page content (not browser UI)
4. Parse and return article titles

### Multi-Step Form Filling

```text
Fill out the registration form with:
Name: John Doe
Email: john@example.com
Phone: 555-0123
```

Claude will:
1. Take a **Snapshot** to locate form fields
2. Use **MultiEdit** to fill all fields at once for efficiency
3. Optionally click the submit button

### File Operations

```text
Find all .log files on my desktop and move them to a folder called "logs"
```

Claude will:
1. Use **FileSystem** with `search` mode to find `.log` files
2. Use **FileSystem** with `copy` or `move` mode to organize them
3. Confirm the operation completed

## Tips for Effective Usage

<Warning>
  **Important**: Windows-MCP has full system access. Be careful when asking Claude to delete files, modify the registry, or terminate processes. Always review actions before confirming destructive operations.
</Warning>

<CardGroup cols={2}>
  <Card title="Be Specific" icon="crosshairs">
    Provide clear instructions with exact names, paths, and values. Instead of "open the browser", say "open Chrome" or "open Firefox".
  </Card>
  
  <Card title="Use Natural Language" icon="message">
    You don't need to specify which tools to use—Claude will choose the right tools based on your request.
  </Card>
  
  <Card title="Review Before Acting" icon="eye">
    For destructive operations, Claude may ask for confirmation. Review the proposed actions carefully.
  </Card>
  
  <Card title="Check Snapshots" icon="camera">
    If automation isn't working as expected, ask Claude to "take a snapshot" to see what it observes on your desktop.
  </Card>
</CardGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Tools not appearing in Claude Desktop">
    1. Verify `claude_desktop_config.json` is saved correctly
    2. Restart Claude Desktop completely
    3. Check Claude's developer console for errors (View > Developer > Toggle Developer Tools)
    4. Ensure UV is installed and accessible: `uvx --version`
  </Accordion>
  
  <Accordion title="Server timeout on first run">
    This is normal! Windows-MCP installs dependencies on first run, which takes 1-2 minutes. Simply restart Claude Desktop and it should work.
  </Accordion>
  
  <Accordion title="Application names not recognized">
    Windows-MCP launches apps via the Start menu. Use the common name you'd type in the Start menu search (e.g., "Notepad", "Word", "Chrome").
    
    If an app doesn't launch, try:
    - Using the exact Start menu name
    - Disabling the App tool if your Windows language is not English
    - Using PowerShell tool to launch via command: `Start-Process "path\to\app.exe"`
  </Accordion>
  
  <Accordion title="Click/Type not working on specific elements">
    1. Take a snapshot with `use_vision=True` to see what Claude observes
    2. Check if the element has a label/ID in the interactive elements list
    3. Try clicking by coordinates instead of label
    4. Some applications block UI automation—try using keyboard shortcuts instead
  </Accordion>
</AccordionGroup>

## Disabling Telemetry

By default, Windows-MCP collects anonymous usage data (tool names and errors only—no arguments or outputs). To disable:

```json
{
  "mcpServers": {
    "windows-mcp": {
      "command": "uvx",
      "args": [
        "windows-mcp"
      ],
      "env": {
        "ANONYMIZED_TELEMETRY": "false"
      }
    }
  }
}
```

Save and restart Claude Desktop.
