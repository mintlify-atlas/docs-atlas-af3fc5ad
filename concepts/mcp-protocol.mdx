Windows-MCP is a **Model Context Protocol (MCP) server** that exposes Windows desktop automation capabilities to AI agents. It uses [FastMCP](https://github.com/jlowin/fastmcp) for protocol implementation and transport handling.

## What is MCP?

The **Model Context Protocol** is a standard for connecting AI agents with external tools and data sources. It defines:

- **Tools**: Functions that AI agents can invoke (e.g., `Click`, `Type`, `Snapshot`)
- **Resources**: Data sources that agents can read (not used in Windows-MCP)
- **Prompts**: Predefined instructions or templates (not used in Windows-MCP)
- **Sampling**: Request LLM completions through the client (used in `Scrape` tool)

<Info>
MCP separates **capability** (what tools are available) from **policy** (when to use them). The LLM client decides which tools to invoke based on the user's request and tool descriptions.
</Info>

## FastMCP Integration

Windows-MCP uses [FastMCP](https://github.com/jlowin/fastmcp) to handle MCP protocol implementation, transport layer, and message serialization.

### Server Initialization

<CodeGroup>
```python Basic Setup
from fastmcp import FastMCP, Context
from contextlib import asynccontextmanager

# Define lifespan for resource management
@asynccontextmanager
async def lifespan(app: FastMCP):
    global desktop, watchdog, analytics
    
    # Startup: initialize services
    desktop = Desktop()
    watchdog = WatchDog()
    if os.getenv("ANONYMIZED_TELEMETRY", "true").lower() != "false":
        analytics = PostHogAnalytics()
    
    watchdog.set_focus_callback(desktop.tree.on_focus_change)
    
    try:
        watchdog.start()
        await asyncio.sleep(1)  # Allow watchdog to initialize
        yield  # Server runs
    finally:
        # Shutdown: cleanup resources
        if watchdog:
            watchdog.stop()
        if analytics:
            await analytics.close()

# Create FastMCP instance
mcp = FastMCP(
    name="windows-mcp",
    instructions="Windows MCP server provides tools to interact directly with the Windows desktop",
    lifespan=lifespan
)
```

```python Run Server
# Entry point
if __name__ == "__main__":
    # Local mode: stdio transport (default)
    mcp.run(transport="stdio", show_banner=False)
    
    # Network mode: SSE or streamable-http
    mcp.run(
        transport="sse",
        host="localhost",
        port=8000,
        show_banner=False
    )
```
</CodeGroup>

<Accordion title="Why FastMCP?">
FastMCP provides:
- **Declarative API**: Register tools using Python decorators
- **Type Safety**: Automatic parameter validation from type hints
- **Multiple Transports**: stdio, SSE, streamable-http
- **Lifecycle Management**: Async context manager for startup/shutdown
- **Context Injection**: Access to client context for sampling

This eliminates boilerplate and lets Windows-MCP focus on desktop automation logic.
</Accordion>

## Tool Registration

### Decorator-Based Registration

Tools are registered using the `@mcp.tool()` decorator. FastMCP introspects the function signature to generate tool schemas automatically.

<CodeGroup>
```python Simple Tool
@mcp.tool(
    name="Click",
    description="Performs mouse clicks at specified coordinates [x, y] or passing a UI element's label/id.",
    annotations=ToolAnnotations(
        title="Click",
        readOnlyHint=False,      # Tool modifies state
        destructiveHint=True,    # Action is not undoable
        idempotentHint=False,    # Multiple calls may differ
        openWorldHint=False,     # Does not access external resources
    ),
)
@with_analytics(analytics, "Click-Tool")
def click_tool(
    loc: list[int] | None = None,
    label: int | None = None,
    button: Literal["left", "right", "middle"] = "left",
    clicks: int = 1,
    ctx: Context = None,
) -> str:
    """Tool implementation"""
    if loc is None and label is None:
        raise ValueError("Either loc or label must be provided.")
    
    if label is not None:
        # Resolve label to coordinates
        if desktop.desktop_state is None:
            raise ValueError("Desktop state is empty. Please call Snapshot first.")
        loc = list(desktop.get_coordinates_from_label(label))
    
    # Execute action
    desktop.click(loc=loc, button=button, clicks=clicks)
    
    # Return human-readable result
    num_clicks = {0: "Hover", 1: "Single", 2: "Double"}
    return f"{num_clicks.get(clicks)} {button} clicked at ({loc[0]},{loc[1]})."
```

```python Async Tool with Sampling
@mcp.tool(
    name="Scrape",
    description="Fetch/scrape web page content from a URL.",
    annotations=ToolAnnotations(
        title="Scrape",
        readOnlyHint=True,
        destructiveHint=False,
        idempotentHint=True,
        openWorldHint=True,  # Accesses external URLs
    ),
)
@with_analytics(analytics, "Scrape-Tool")
async def scrape_tool(
    url: str,
    query: str | None = None,
    use_dom: bool | str = False,
    use_sampling: bool | str = True,
    ctx: Context = None,
) -> str:
    use_dom = use_dom is True or (isinstance(use_dom, str) and use_dom.lower() == "true")
    use_sampling = use_sampling is True or (isinstance(use_sampling, str) and use_sampling.lower() == "true")
    
    # Fetch content (HTTP or DOM extraction)
    if not use_dom:
        content = desktop.scrape(url)
    else:
        # Extract from active browser tab
        desktop_state = desktop.get_state(use_vision=False, use_dom=True)
        tree_state = desktop_state.tree_state
        if not tree_state.dom_node:
            return f"No DOM information found. Please open {url} in browser first."
        content = "\n".join([node.text for node in tree_state.dom_informative_nodes])
    
    # Use LLM to summarize if sampling enabled
    if use_sampling and ctx is not None:
        try:
            focus = f" Focus specifically on: {query}." if query else ""
            result = await ctx.sample(
                messages=f"Raw scraped content from {url}:\n\n{content}",
                system_prompt=(
                    "You are a web content extractor. Given raw webpage content, extract and present "
                    "only the meaningful information in clean, concise prose or structured format. "
                    f"Strip out navigation menus, cookie banners, ads, footer links.{focus}"
                ),
                max_tokens=2048,
            )
            return f"URL: {url}\nContent:\n{result.text}"
        except Exception:
            pass  # Fall through to raw content
    
    return f"URL: {url}\nContent:\n{content}"
```
</CodeGroup>

### Tool Annotations

Annotations provide **metadata** about tool behavior to help clients make informed decisions:

| Annotation | Meaning | Example |
|------------|---------|----------|
| `readOnlyHint` | Tool does not modify state | `Snapshot` (True), `Click` (False) |
| `destructiveHint` | Action is irreversible | `Shell` (True), `Move` (False) |
| `idempotentHint` | Multiple calls produce same result | `Scroll` (True), `App` (False) |
| `openWorldHint` | Accesses external resources | `Scrape` (True), `Click` (False) |

<Note>
These are **hints**, not enforced contracts. Clients may use them for safety checks or UI indicators.
</Note>

## Parameter Handling

### Type Coercion

MCP passes parameters as **JSON values**. FastMCP coerces them to Python types based on function signatures, but some MCP clients send unexpected types.

Windows-MCP handles string booleans defensively:

```python
def type_tool(
    text: str,
    clear: bool | str = False,  # Accept bool or string
    press_enter: bool | str = False,
    ctx: Context = None,
) -> str:
    # Normalize to bool
    clear = clear is True or (isinstance(clear, str) and clear.lower() == 'true')
    press_enter = press_enter is True or (isinstance(press_enter, str) and press_enter.lower() == 'true')
    
    # Use normalized values
    if clear:
        uia.SendKeys("{Ctrl}a{Back}", waitTime=0.05)
    
    uia.SendKeys(text, interval=0.02)
    
    if press_enter:
        uia.SendKeys("{Enter}", waitTime=0.05)
```

### Optional Parameters with Unions

```python
def click_tool(
    loc: list[int] | None = None,  # Either provide coordinates
    label: int | None = None,      # Or provide label ID
    button: Literal["left", "right", "middle"] = "left",
    clicks: int = 1,
    ctx: Context = None,
) -> str:
    if loc is None and label is None:
        raise ValueError("Either loc or label must be provided.")
    
    # One of them is guaranteed to be non-None
    if label is not None:
        loc = list(desktop.get_coordinates_from_label(label))
    
    desktop.click(loc=loc, button=button, clicks=clicks)
    return f"Clicked at ({loc[0]},{loc[1]})"
```

## Return Values

Tools return **strings** for simplicity. Complex data is formatted as human-readable text.

<CodeGroup>
```python Simple String
def wait_tool(duration: int, ctx: Context = None) -> str:
    time.sleep(duration)
    return f"Waited for {duration} seconds."
```

```python Structured Text
def state_tool(use_vision: bool = False, ctx: Context = None):
    desktop_state = desktop.get_state(use_vision=use_vision)
    
    interactive_elements = desktop_state.tree_state.interactive_elements_to_string()
    scrollable_elements = desktop_state.tree_state.scrollable_elements_to_string()
    windows = desktop_state.windows_to_string()
    
    return [dedent(f'''
    Cursor Position: {desktop_state.cursor_position}
    Screenshot Resolution: {desktop_state.screenshot_size.to_string()}
    
    Focused Window:
    {desktop_state.active_window_to_string()}
    
    Opened Windows:
    {windows}
    
    List of Interactive Elements:
    {interactive_elements or "No interactive elements found."}
    
    List of Scrollable Elements:
    {scrollable_elements or "No scrollable elements found."}
    ''')]
```

```python Text + Image
def state_tool(use_vision: bool = True, ctx: Context = None):
    desktop_state = desktop.get_state(use_vision=True)
    
    # Convert PIL Image to bytes
    buffered = io.BytesIO()
    desktop_state.screenshot.save(buffered, format="PNG")
    screenshot_bytes = buffered.getvalue()
    buffered.close()
    
    # Return list: [text, image]
    return [
        f"Cursor Position: {desktop_state.cursor_position}\n...",
        Image(data=screenshot_bytes, format='png')
    ]
```
</CodeGroup>

<Accordion title="Why Not JSON?">
Returning structured JSON requires clients to parse and format it themselves. Human-readable text is universally understood by LLMs and easier to debug.

For `Snapshot`, returning both text (element list) and image (screenshot) provides the best of both worlds:
- Non-vision models use element list
- Vision models can see the actual UI
</Accordion>

## Context & Sampling

The `ctx: Context` parameter provides access to **client capabilities**.

### LLM Sampling

The `Scrape` tool uses `ctx.sample()` to request an LLM completion from the client:

```python
@mcp.tool(name="Scrape", ...)
async def scrape_tool(
    url: str,
    query: str | None = None,
    use_sampling: bool | str = True,
    ctx: Context = None,
) -> str:
    content = desktop.scrape(url)  # Fetch raw HTML
    
    if use_sampling and ctx is not None:
        try:
            result = await ctx.sample(
                messages=f"Raw scraped content from {url}:\n\n{content}",
                system_prompt="You are a web content extractor. Extract only meaningful information...",
                max_tokens=2048,
            )
            return f"URL: {url}\nContent:\n{result.text}"
        except Exception:
            pass  # Fallback to raw content
    
    return f"URL: {url}\nContent:\n{content}"
```

<Note>
Sampling is **optional** and may not be supported by all clients. Always provide a fallback.
</Note>

## Transport Modes

FastMCP supports multiple transports. Windows-MCP exposes them via CLI flags:

<Tabs>
<Tab title="stdio (Default)">
**Use Case**: Direct connection from MCP clients (Claude Desktop, Cursor, etc.)

```bash
uvx windows-mcp
# or
uvx windows-mcp --transport stdio
```

**Protocol**: JSON-RPC over stdin/stdout

**Pros**:
- Simple configuration
- Low latency
- Process isolation

**Cons**:
- Only one client per server process
- No network access
</Tab>

<Tab title="SSE">
**Use Case**: Network-accessible server for web clients

```bash
uvx windows-mcp --transport sse --host localhost --port 8000
```

**Protocol**: Server-Sent Events over HTTP

**Pros**:
- Multiple clients
- Browser-friendly
- Firewall-friendly (HTTP only)

**Cons**:
- Higher latency than stdio
- Requires CORS handling for browsers
</Tab>

<Tab title="Streamable HTTP">
**Use Case**: Production deployments, remote mode

```bash
uvx windows-mcp --transport streamable-http --host localhost --port 8000
```

**Protocol**: HTTP streaming (chunked transfer encoding)

**Pros**:
- Most reliable for production
- Load balancer compatible
- Works behind proxies

**Cons**:
- Slightly more complex than SSE
</Tab>
</Tabs>

### Transport Selection Logic

```python
@click.command()
@click.option(
    "--transport",
    type=click.Choice(["stdio", "sse", "streamable-http"]),
    default="stdio"
)
@click.option("--host", default="localhost", type=str)
@click.option("--port", default=8000, type=int)
def main(transport, host, port):
    config = Config(
        mode=os.getenv("MODE", "local").lower(),
        sandbox_id=os.getenv("SANDBOX_ID", ''),
        api_key=os.getenv("API_KEY", '')
    )
    
    match config.mode:
        case "local":
            match transport:
                case "stdio":
                    mcp.run(transport="stdio", show_banner=False)
                case "sse" | "streamable-http":
                    mcp.run(
                        transport=transport,
                        host=host,
                        port=port,
                        show_banner=False
                    )
        case "remote":
            # Proxy mode: connect to windowsmcp.io
            client = AuthClient(api_key=config.api_key, sandbox_id=config.sandbox_id)
            client.authenticate()
            backend = StreamableHttpTransport(
                url=client.proxy_url,
                headers=client.proxy_headers
            )
            proxy_mcp = FastMCP.as_proxy(ProxyClient(backend), name="windows-mcp")
            proxy_mcp.run(transport=transport, host=host, port=port, show_banner=False)
```

## Analytics Wrapper

Windows-MCP uses a decorator to track tool usage:

```python
from functools import wraps

def with_analytics(analytics_client, tool_name: str):
    """Decorator to wrap tool functions with analytics tracking"""
    def decorator(func):
        @wraps(func)
        async def async_wrapper(*args, **kwargs):
            try:
                result = await func(*args, **kwargs)
                if analytics_client:
                    analytics_client.track_tool_success(tool_name)
                return result
            except Exception as e:
                if analytics_client:
                    analytics_client.track_tool_error(tool_name, str(type(e).__name__))
                raise
        
        @wraps(func)
        def sync_wrapper(*args, **kwargs):
            try:
                result = func(*args, **kwargs)
                if analytics_client:
                    analytics_client.track_tool_success(tool_name)
                return result
            except Exception as e:
                if analytics_client:
                    analytics_client.track_tool_error(tool_name, str(type(e).__name__))
                raise
        
        return async_wrapper if asyncio.iscoroutinefunction(func) else sync_wrapper
    return decorator

# Usage
@mcp.tool(name="Click", ...)
@with_analytics(analytics, "Click-Tool")
def click_tool(...):
    pass
```

**What's Tracked**:
- Tool invocation count (success/failure)
- Error type (e.g., `ValueError`, `LookupError`)

**What's NOT Tracked**:
- Tool arguments
- Tool outputs
- User data

Disable with `ANONYMIZED_TELEMETRY=false`.

## Tool Registry

Windows-MCP exposes **15 tools** across 5 categories:

### Input Tools
- **Click**: Mouse clicks (single, double, right, middle)
- **Type**: Keyboard input with caret control
- **Scroll**: Vertical/horizontal scrolling
- **Move**: Cursor movement and drag-and-drop
- **Shortcut**: Keyboard shortcuts (Ctrl+C, Alt+Tab, etc.)
- **MultiSelect**: Select multiple items (Ctrl+Click)
- **MultiEdit**: Batch text input

### Window Management
- **App**: Launch, resize, switch applications
- **Snapshot**: Capture desktop state (UI tree + screenshot)

### System Tools
- **Shell**: Execute PowerShell commands
- **FileSystem**: File operations (read, write, copy, move, delete, list, search)
- **Process**: List and kill processes
- **Clipboard**: Read/write clipboard
- **Registry**: Access Windows Registry

### Web Tools
- **Scrape**: Fetch web page content with LLM summarization

### Utility Tools
- **Wait**: Pause execution
- **Notification**: Send Windows toast notification

See [Tools Reference](/tools/overview) for detailed documentation on each tool.

## Error Handling

Tools should return **user-friendly error messages** rather than throwing exceptions when possible:

```python
@mcp.tool(name="Click", ...)
def click_tool(loc: list[int] | None = None, label: int | None = None, ...) -> str:
    # Validation errors: raise ValueError for missing required params
    if loc is None and label is None:
        raise ValueError("Either loc or label must be provided.")
    
    # State errors: raise ValueError with actionable message
    if label is not None and desktop.desktop_state is None:
        raise ValueError("Desktop state is empty. Please call Snapshot first.")
    
    # Execution errors: return error string
    try:
        desktop.click(loc=loc, button=button, clicks=clicks)
        return f"Clicked at ({loc[0]},{loc[1]})"
    except Exception as e:
        return f"Click failed: {str(e)}"
```

<Note>
**Why not always raise exceptions?** Returning error strings lets the LLM see what went wrong and adapt its strategy. Raising exceptions terminates the tool execution immediately.
</Note>

## Best Practices

1. **Descriptive Tool Names**: Use action verbs (`Click`, `Type`, `Scrape`)
2. **Rich Descriptions**: Explain what the tool does, when to use it, and key parameters
3. **Type Hints**: FastMCP uses them for schema generation and validation
4. **Defensive Parsing**: Handle string booleans, optional unions, and malformed inputs
5. **Human-Readable Returns**: Format output as clear, structured text
6. **Error Context**: Explain what went wrong and how to fix it
7. **Annotations**: Set appropriate hints for safety and performance
8. **Async When Needed**: Use `async def` only if the tool does I/O or sampling
